    • Local Business Schema

    • About the project

      Help search engines know your business better

    • Background

    • As a website building platform, it was essential for Duda to support schema. As most websites represent businesses, embedding local business schema.

    • Roles and Responsibilities

    • I was the product manager and designer, working with one full stack developer and manual QA engineer.

    • Target Audience

    • Our target audience was very broad, every site that represents a business - meaning the majority of our users.

    • High Level Goals

    • Our goal was to help users create local business schema for their sites with the minimum effort required.

    • The Solution

    • Schema is markup that helps search engines understand information about your website. Local business schema is one type of Schema which focuses on the type of business the website represents.
      Local business schema based on Content Library Data. The content library collects data entered from widgets across the sites, as well as can contain basic information about your site. Our solution was to use this pre-existing data and the content library form-like interface to populate schema and allow users to generate it with a click of a button.

    • 02

      The Process

    • Competitor Research & Interviews

    • I dived into the schema world and researched this topic, as well as what are the best practices for embedding schema in your sites. There were two practices - website builders directed users to schema generators and required them to generate schema outside the platform, copy the code and paste it in the platform. WP plugins and other softwares made schema more accessible by layout out all the required fields. I also talked to some clients who were after this feature, to understand what type of schema they are after.

    • Field Types

    • We embedded the recommended fields by Google’s guidelines for local business schema and mapped them to the built in content library fields.

    • High Level Flows

    • Unlike other complex features, Local Busienss Schema is only a toggle. The user can turn it on, review data, fix data and review again, validate and enable Local Business Schema for their site.

    • 03

      Key Features

    • Read only popup based on existing fields

    • For the first release we created a simple popup that allows users to review their existing schema data, validate it with google and enable schema.
      The plan for the next release was initially to support editing from that popup, but this read-only popup was adopted by users who were very pleased with it, so we skipped the editing from the popup phase and continued to the next one.

    • Preview popup that was released

    • Planned popup that included editing (never released)

    • Custom fields

      Local Business Schema supports ~150 business types, each one requires different fields. When a user selected a business type, we suggested a custom field according to the selected business type.

    • Rich Results Validation

      We allow users to validate that their schema is eligible to schema with a click of a button. This lets them make sure they filled in the right data before publishing the site and generating schema.

    • Multi-location schema - show how more than 10 locations work?

      Since business info was based on location - where each location had its data - I came up with a friendlier structure to handle and manage locations. According to our data, most people have up to 4 locations but we have a few clients that could have up to 10.
      That's why we had to address two different scenarios. Each location will be represented as a tab, where all the location data will be displayed [map, name, email, phone, etc]. If there are more than 10 locations the arrow will turn into a dropdown for easier navigation between the different locations.

    • 04

      Releases

    • Scope

    • This feature was released in a month. We had a deadline so we went for the fastest solution. Afterwards we added business name, custom fields and other improvements.

    • Release Phases

    • This feature was gradually release due to different clients with different needs.
      - API first
      - UI - read only, closed amount of fields
      - UI + API - more business types
      - UI - edit (canceled)
      - UI + API - Add custom fields

    • Impact

    • This feature was happily recieved by our users as 'no-code schema'. We had more than 2K accounts and 25K sites that were using this. The majority of the sites were activated by API.
